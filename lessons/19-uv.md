## Introduction

### What this lesson is about

As your programs grow, copying and pasting functions between files becomes hard
to manage. This lesson introduces **modules** — Python files you can import to
reuse code safely and clearly. You will learn how to import built-in modules,
write your own module, and use the `if __name__ == "__main__":` entry point.

### Why you need this

Modules are one of the most important ideas in Python. They let you:
- avoid repeating code
- keep files small and focused
- test and improve one part without breaking everything
- share useful functions across multiple scripts

---

## Do

### Step 1 — Import a built-in module

```python
import math

radius = 4
area = math.pi * radius**2
print(area)
```

`math` is a built-in module from the Python standard library. Importing a
module gives your script access to functions and constants from that module.

---

### Step 2 — Write your own module

Create a file called `math_utils.py`:

```python
def circle_area(radius):
    return 3.14159 * radius**2


def metres_to_cm(metres):
    return metres * 100
```

Then create another file called `app.py`:

```python
import math_utils

print(math_utils.circle_area(4))
print(math_utils.metres_to_cm(1.25))
```

Now `app.py` can reuse functions from `math_utils.py` without copying code.

---

### Step 3 — Module file structure and entry points

A common beginner layout is:

```text
project/
├── app.py
└── math_utils.py
```

- `math_utils.py` holds reusable functions.
- `app.py` is the script entry point for running the program.

When you run `python app.py`, Python executes `app.py` first, then imports
`math_utils.py`.

---

### Step 4 — `if __name__ == "__main__":`

Add this to the bottom of `math_utils.py`:

```python
def circle_area(radius):
    return 3.14159 * radius**2


def metres_to_cm(metres):
    return metres * 100


if __name__ == "__main__":
    print("Running quick module checks...")
    print(circle_area(2))
    print(metres_to_cm(3))
```

This block runs **only** when `math_utils.py` is run directly:

```bash
python math_utils.py
```

It does **not** run when `math_utils` is imported by `app.py`. This lets you
keep quick tests/examples in the module without affecting importing scripts.

---

### Step 5 — Two import styles

```python
import math_utils
print(math_utils.circle_area(5))
```

```python
from math_utils import circle_area
print(circle_area(5))
```

Both work. The first style is often clearer for beginners because it shows
exactly where the function came from.

---

### Step 6 — Why modules matter

If two scripts need the same grade-calculation function, put it once in a
module and import it into both scripts. If the grade boundary changes, you
update one file instead of many.

## Explore

1. Create a new module `grade_utils.py` containing a `grade(score)` function.
   Import it into a second file and use it for a list of scores.
2. Add a `__main__` block to `grade_utils.py` that prints example grades. What
   happens when you run `grade_utils.py` directly vs importing it?
3. Change one function name in your module and update imports. What error do
   you get if you forget to update one import?
4. Write two scripts that both import the same conversion function module
   (e.g. kilometres to miles). Why is this safer than copy/paste?
5. Try both `import module` and `from module import function` in your own
   script. Which style do you find clearer and why?
